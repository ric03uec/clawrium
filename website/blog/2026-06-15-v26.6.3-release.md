The v26.6.3 release brings significant improvements to agent management, provider configuration, and GUI experience, with new features for ethos agents and enhanced provider support.

<!-- truncate -->

## What changed

**Hermes multi-provider attach** with --role and detach-primary guard. Now you can attach multiple providers to a hermes agent with specific roles, and the detach-primary guard ensures you don't accidentally remove the primary provider.

**Jinja2 promoted to runtime dependency**. This change ensures that all hermes agents have consistent template rendering capabilities without requiring manual installation of jinja2.

**Auxiliary blocks for hermes multi-provider attachments**. The render system now properly emits auxiliary blocks for multi-provider configurations, enabling more complex agent setups.

**configure_agent routing through render_hermes**. The configure_agent function has been refactored to use render_hermes, eliminating legacy templates and improving code maintainability.

**Ethos as a first-class agent type**. Ethos agents are now fully supported as a first-class agent type, with dedicated management and configuration options.

**Local skill lifecycle for agents**. The local skill lifecycle has been implemented in three phases: Phase A (local skill helpers), Phase B (agent-local skill lifecycle), and Phase C (per-agent local skill lifecycle in GUI). This enables agents to have their own persistent skills that are not shared across the fleet.

**Ethos agent type with declarative fleet management**. Ethos agents now support declarative fleet management, allowing you to define and manage fleets of ethos agents through configuration files.

**Bedrock multi-attach with shared AWS credentials**. You can now configure multiple Bedrock providers to share the same AWS credentials, simplifying configuration and reducing credential management overhead.

**Providers page UX rework**. The Providers page has been completely redesigned with a new interface for managing providers, including better integration with Bedrock AWS credential flow.

**Providers page follow-ups**. Improvements to the Providers page including registry dropdown, fleet-style table, and clearer button labels for better usability.

**Sidebar reorg: Agents tab and coming-soon rows**. The sidebar has been reorganized with a dedicated Agents tab and new coming-soon (i)/Upvote rows to better organize features and user feedback.

**Sidebar coming-soon modal and Settings to footer**. A new coming-soon modal has been added to the sidebar, and Settings have been moved to the footer for a cleaner interface.

**Litellm provider type added**. The litellm provider type is now available, allowing you to use the litellm API for inference with a wide range of models.

## Why

The hermes multi-provider attach improvements address the need for more flexible and secure provider management, with the detach-primary guard preventing accidental misconfigurations.

The jinja2 dependency change ensures consistent template rendering across all agents, reducing configuration issues.

The local skill lifecycle implementation enables agents to have their own persistent skills, improving personalization and reducing conflicts in multi-agent environments.

The ethos agent enhancements provide a more robust and configurable agent type, with declarative fleet management making it easier to scale and manage ethos agents.

The provider UI improvements (Providers page, sidebar reorg) were driven by user feedback to make provider management more intuitive and user-friendly.

The litellm provider addition expands the range of available models and reduces vendor lock-in for users.

## Try it

```bash
# Install or upgrade
uv tool install clawrium@v26.6.3

# Try the new features
clawctl agent get
clawctl agent describe <name>
clawctl agent open <name>

# For ethos agents
clawctl agent create --type ethos my-ethos-agent
clawctl agent get --type ethos

# For multi-provider hermes agents
clawctl agent configure hermes-agent --provider bedrock --role primary
clawctl agent configure hermes-agent --provider openai --role secondary
```
