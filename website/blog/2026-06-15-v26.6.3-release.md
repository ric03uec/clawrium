The v26.6.3 release recovers from a failed v26.6.2 cut, with a critical fix to the GUI's provider management flow and a new litellm provider type.

<!-- truncate -->

## What changed

**v26.6.3 recovery** — This release corrects a critical issue in the v26.6.2 cut that caused the Providers page to fail to load. The fix ensures the GUI's provider configuration state is properly initialized and persisted.

**litellm provider** — A new provider type is now available, enabling access to a wide range of LLMs through the litellm unified API. This provides a single interface to models from OpenAI, Anthropic, Google, and more, with consistent configuration and credential management.

**Provider page UX** — The Providers page has been restructured to use a fleet-style table, with clear row actions and a dedicated button for adding new providers. The UI now better reflects the multi-provider model and makes it easier to manage multiple API keys.

**Sidebar reorganization** — The main sidebar has been restructured to group related items. The "Agents" tab is now the primary entry point, and a new "Coming Soon" section has been added to highlight upcoming features and allow users to upvote them.

## Why

The v26.6.2 release contained a critical bug that broke the provider management UI, making it impossible to configure new LLM providers. This hotfix ensures the system is stable and usable.

The addition of the litellm provider type significantly expands the range of available models, making it easier for users to experiment with different LLMs without managing multiple API keys and configurations.

The UI improvements to the Providers page and sidebar are based on user feedback and aim to make the interface more intuitive and easier to navigate.

## Try it

```bash
# Install or upgrade
uv tool install clawrium@v26.6.3

# Start the GUI and configure your providers
clawctl agent start <name>

# Add a new litellm provider
clawctl agent configure <name> --provider litellm
```
