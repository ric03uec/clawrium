The v26.6.3 release brings critical fixes to the agent lifecycle, improved provider configuration, and a new `clawctl agent doctor` command to diagnose and resolve common issues.

<!-- truncate -->

## What changed

**Agent lifecycle stability** — `clawctl agent configure`, `sync`, and `restart` now always re-pair the gateway with a fresh bearer token, even if the agent is already running. This fixes silent 401s in remote `clawctl agent chat` sessions after a daemon restart. (See #437)

**Provider configuration validation** — The `clawctl agent configure` command now validates provider settings before writing the config file, preventing invalid or incomplete provider entries from being saved. This includes checks for required fields, valid model names, and network reachability.

**`clawctl agent doctor`** — A new diagnostic command that scans the agent's state, checks for known misconfigurations, and suggests fixes. Run `clawctl agent doctor <name>` to get a report of potential issues and how to resolve them.

**Improved error messages** — Error output from `clawctl` and agent daemons is now more user-friendly, with clear action items and links to documentation.

**Documentation updates** — The `docs/installation.md` and `docs/host-preparation.md` files have been updated to reflect the new `clawctl` commands and configuration flow. The website docs have been automatically mirrored.

## Why

The agent lifecycle fixes address a long-standing issue where remote chat sessions would break after a daemon restart, requiring users to manually re-pair. The new `doctor` command was requested by several users who were struggling to diagnose why their agents weren't responding.

## Try it

```bash
# Install or upgrade
uv tool install clawrium@v26.6.3

# Check your agent's health
clawctl agent doctor <name>

# Reconfigure with validation
clawctl agent configure <name>

# Restart to trigger re-pairing
clawctl agent restart <name>
```
