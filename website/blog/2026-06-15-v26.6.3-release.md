The v26.6.3 release brings critical fixes to the agent lifecycle, improved provider configuration, and a new `clawctl agent doctor` command to diagnose and resolve common issues.

<!-- truncate -->


## What changed

**BREAKING:** The legacy `clm` binary has been removed. Switch from `clm <command>` to `clawctl <command>` — the argument surface is unchanged.

**Added:** The `litellm` provider now supports direct integration with any OpenAI-compatible proxy (LiteLLM, vLLM, etc.), allowing teams to route requests through their existing infrastructure without maintaining a parallel direct-API path.

**Added:** Multi-attach now supports sharing a single provider credential set (access_key, secret_key, region) across multiple attachment slots on a single hermes agent.

**Changed:** The Dashboard sidebar has been reorganized. A new **Agents** entry now hosts the fleet table. **Settings** has been moved from the main nav into the footer. **MCPs**, **Scheduled Jobs**, and **Agent Builder** now render as grayed-out "coming soon" buttons that open a modal with Upvote/Discord/Feature-request actions.

**Fixed:** Error output from `clawctl` and agent daemons is now more user-friendly, with clear action items and links to documentation.

**Documentation updates** — The `docs/installation.md` and `docs/host-preparation.md` files have been updated to reflect the new `clawctl` commands and configuration flow. The website docs have been automatically mirrored.


## Why

The removal of `clm` consolidates the CLI interface into a single, consistent tool (`clawctl`), reducing user confusion and maintenance overhead. The `litellm` provider unblocks teams already running an OpenAI-compatible proxy from having to maintain a parallel direct-API path. Multi-attach with shared credentials simplifies credential management for teams using AWS Bedrock across multiple agent instances. The new **Agents** dashboard entry provides a unified view of all agents, replacing the fragmented layout from previous versions.


## Try it

```bash
# Install or upgrade
uv tool install clawrium@v26.6.3

# Check your agent's health
clawctl agent doctor <name>

# Reconfigure with validation
clawctl agent configure <name>

# Restart to trigger re-pairing
clawctl agent restart <name>
```
